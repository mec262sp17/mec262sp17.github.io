What to do when data is not linear?

Kernel trick¶

Not always possible for data to fit a line or be linearly separable.

  • Apply nonlinear transformation on the data and then apply linear fit

import numpy as np
import matplotlib.pyplot as plt
%matplotlib inline

pi = 3.14159
x = 10*np.linspace(0,10,1001)
y = x + 10*np.sin(.2*x+pi/3) + 2*np.random.randn(1001)
plt.plot(x,y,'r.')
plt.xlabel('X')
plt.ylabel('Y');

Assume¶

$$ y = a x + b sin(x) + c cos(x) $$

as \( y \) is linear in unknowns, we can compute \( a,b \) and \( c \) using linear regression on \(x \), sin\( x \) and \( cos(x) \).

from sklearn import linear_model

x_feat = [[v,np.sin(.2*v),np.cos(.2*v)] for v in x]
y_feat = [v for v in y]
reg_kernel = linear_model.LinearRegression()
reg_kernel.fit(x_feat,y_feat)
y_pred_kernel = reg_kernel.predict(x_feat)

x_feat_lin = [[v] for v in x]
reg_linear = linear_model.LinearRegression()
reg_linear.fit(x_feat_lin,y_feat)
y_pred_linear = reg_linear.predict(x_feat_lin)

Kernel trick for classification¶

Recall, SVM divides planes into two parts with a line, how to do so for nonlinear functions?

In [6]:
from sklearn import svm
# Our dataset and targets
X = np.c_[(.4, -.7),(-1.5, -1),(-1.4, -.9),(-1.3, -1.2),
          (-1.1, -.2),(-1.2, -.4),(-.5, 1.2),(-1.5, 2.1),(1, 1),
          # --
          (1.3, .8),(1.2, .5),(.2, -2),(.5, -2.4),(.2, -2.3),(0, -2.7),(1.3, 2.1)].T
Y = [0] * 8 + [1] * 8

svm_linear = svm.SVC(kernel='linear', gamma=2)
svm_linear.fit(X, Y)

x_min,x_max,y_min,y_max = -3,3,-3,3
XX, YY = np.mgrid[x_min:x_max:200j, y_min:y_max:200j]
Z = svm_linear.decision_function(np.c_[XX.ravel(), YY.ravel()])
Z = Z.reshape(XX.shape)

  • Non-linear methods carve out regions in feature space where each region belongs to one type of variable.

Alternative approach.

  • Combine multiple linear classifiers to carve out region in the space.
  • Recall \( y = mx + c \) separates plane into two parts one where \( y > mx + c \) and another where \( y < mx + c \)
